Figma — Design panel (right sidebar)

  • Sezioni collassabili: Layout, Fill, Stroke — non una riga per proprietà.
  • In Layout: griglia allineamento 3×3 + coppie X/Y e W/H con prefisso lettera nel campo.
  • Stroke = colore + spessore sulla stessa riga; effetti opzionali sotto.
  • Contesto selezione in alto (nome layer).
Documentazione Figma →

Cosa portare nel label designer

  • Raggruppare Posizione (XYWH + allinea in etichetta) in un blocco “Layout”.
  • Contorno testo = sezione “Stroke” (toggle + spessore + colore inline).
  • Sfondo = sezione “Fill” con swatch + opacità.
  • Niente sync ↻ per ogni campo — Figma non lo fa; sync thread resta custom vostro.

Webflow — Style panel

  • Sezioni CSS vere: Layout, Spacing, Size, Typography, Backgrounds, Borders, Effects.
  • Spacing widget: diagramma margin/padding — standard de facto per editor visivi.
  • Pallino blu sulla sezione se almeno un valore è impostato.
  • Tipografia: font + size + line height in griglia 2 colonne.
Documentazione Webflow →

Cosa portare nel label designer

  • Padding elementi (T/R/B/L) con widget spacing invece di 4 righe prop-grid.
  • Border / contorno sotto sezione “Borders” con width + color + radius.
  • Typography dedicata — separata da posizione e da contorno.

Canva — Quick actions + Edit panel

  • Canva non usa un pannello proprietà fisso a destra: toolbar contestuale sopra l’elemento.
  • Selezioni testo → font, size, colore, allineamento in barra orizzontale.
  • Pulsante >> apre pannello edit completo (effetti, spaziatura avanzata).
  • Side panel sinistro = contenuti (template, elementi), non proprietà.
Canva Help — editor glow up →

Cosa portare nel label designer

  • Barra floating sopra elemento selezionato: font size, colore, allinea H/V.
  • Sidebar resta per profilo etichetta + add element + sync; props “leggere” on-canvas.
  • Utile su tablet / schermi stretti.

Penpot — Design properties (open source)

  • Stessa famiglia di Figma: sidebar destra “Design properties” dinamica per tipo layer.
  • Proprietà sempre presenti: size, position; opzionali: stroke, shadow, blur.
  • Tipografie salvabili come stili (Assets) — utile per brand kit etichette.
  • Inspect mode separato per dev (come Figma Dev Mode).

BarTender — Properties (software etichette)

  • Pannello destro context-sensitive: cambia con oggetto selezionato.
  • Form intelligenti: tab Attributes · Options · Layout · Style.
  • Label a destra del campo (Windows classic) — familiare per utenti industriali.
  • Gruppi fieldset per sotto-sezioni (Data Source, Font, Border).
BarTender — Properties tabs →

Cosa portare nel label designer

  • Tab nel pannello props: Contenuto | Stile | Layout.
  • Contorno/sfondo/font sotto “Stile”; posizione/dimensioni sotto “Layout”.
  • Più vicino al dominio “etichette” che Figma, meno trendy ma chiaro.

Confronto — stesso dominio, pattern diversi

  • Figma/Penpot: sezioni Design (Layout / Fill / Stroke / Type).
  • Webflow: spacing visuale + sezioni CSS.
  • BarTender: tab + form label-destra (verticali industriali).

Sintesi per Custom Label

  • Utenti creativi / web → direzione Figma.
  • Padding/margin complessi → widget Webflow.
  • Tablet / editing rapido → toolbar Canva + sidebar leggera.
  • Stampa industriale, poche sorprese → tab BarTender.
